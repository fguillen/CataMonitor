A) HTML Structure - top

Header

<div id="header">
		
	<div class="pad">
		
		<h1 id="title"><a href="">Azure</a></h1>
		
		<div id="info">
			
			<div id="infoText">
				Welcome back <a href="javascript:;">John Roberts</a> 
				<br />
				Today is November 15, 2009 17:24 <a href="javascript:;">View Website</a> | 
				<a href="javascript:;">View Latest Activity</a>
			</div> <!-- #infoText -->
			
			<div class="underlay"></div>
		</div>
		
	</div> <!-- .pad -->
	
</div> <!-- #header -->

To change the website title, modify H1#title.

To change the info message and links, modify div#info.

Navigation

<li>
	<a href="javascript:;">Orders</a>
								
	<div class="sub_nav one_col">	
		<div class="col">		
			
			<h3>Help Links</h3>
				
			<ul>
				<li><a href="javascript:;">Section #1</a></li>
				<li><a href="javascript:;">Section #2</a></li>
				<li><a href="javascript:;">Section #3</a></li>
				<li><a href="javascript:;">Section #4</a></li>
			</ul>					
		</div><!-- .col -->
	</div><!-- .sub_nav -->								
</li>

To create a new main navigation link, an LI tag containing desired hyperlink.

To create a sub menu within a navigation link, add a div with the class .sub_nav. Inside of the div.sub_nav you may add up to 3 columns of content by adding a div with the class .col for each column you need.

To customize the content of the column, modify the text in the div.col

Portlets

All the content is contained in blocks with the following structure:

<div class="portlet portlet-closable">	
	<div class="portlet-header">
		<h4>Tabs</h4>
		
		<ul class="portlet-tab-nav">
			<li class="portlet-tab-nav-active"><a href="#tab1">Tab 1</a></li>
			<li><a href="#tab2">Tab 2</a></li>
			<li><a href="#tab3">Tab 3</a></li>
		</ul>
		
		<span class="portlet-toggle-icon"></span>
	</div> <!-- .portlet-header -->	
	
	<div class="portlet-content">			
		<div id="tab1" class="portlet-tab-content">
			<strong>Tab #1</strong>			
			<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod.</p>
		</div> <!-- .portlet-tab-content -->
		
		<div id="tab2" class="portlet-tab-content">
			<strong>Tab #2</strong>			
			<p>Vivamus ac tortor dolor. Donec commodo, dui id ultrices blandit. </p>
		</div> <!-- .portlet-tab-content -->
		
		<div id="tab3" class="portlet-tab-content">
			<strong>Tab #3</strong>			
			<p>Vivamus ac tortor dolor. Donec commodo, dui id ultrices blandit. </p>
		</div> <!-- .portlet-tab-content -->
	</div> <!-- .portlet-content -->	
</div> <!-- .portlet -->

To modify the portlet title, change the H4 text in .portlet-header

To make a portlet closable, add the class .portlet-closable to the div where you added the class .portlet

To create a closed portlet when the page loads, add the class .portlet-state-closed to the same div as .portlet-closable. WARNING: Adding this class to a portlet that is not made closable will hide the content of the portlet permanently.

The portlet toggle icon automatically adds itself when a portlet is made closeable using .portlet-closable

To change the information within a portlet, modify the contents of .portlet-content

To create portlet tabs, add an unordered list with the class .portlet-tab-nav into div.portlet-header. For each tab, the HREF attribute on the anchor tag should be the ID of the DIV containing its content.

To activate a tab upon page load, add the class .portlet-tab-nav-active to the LI tag of the tab you want opened.

To create a tab's content (ex. Tab 1), add a div with the class .portlet-tab-content and a unique ID attribute (ex. id="tab1"). IMPORTANT: The id of this .portlet-tab-content must match the id of the anchor tag.

IMPORTANT: When working with portlet tabs, it is important that all content reside within a div with the class .portlet-tab-content. Any content outside of these will always be visible.

Portlets added within div#main will have the header color of the active theme. Portlets added within div#sidebar will have a grey header color.

Forms

<form class="form label-top" action="" method="get" accept-charset="utf-8">

All styled forms need to have the class .form

To create a form with the labels on top of the fields, add the class .label-top to the form tag. The default for forms is labels to the left of fields and no classes are necessary for this to work.

<div class="field">
	<label for="subject">Subject</label>
	<input type="text" name="subject" value="" id="subject" class="medium" />				
	<span class="field_help">This is the help text for the field</span>
	<span class="field_error">This field is invalid</span>
</div> <!-- .field -->

Specify the label position using the class .label-top to create labels that are on top of the fields. Use the class .label-inline to create labels that are to the left of the fields. These two classes go on the main form tag.

Input elements can contain the classes .small, .medium, .large to specify the size of the an input field.

Specify field help text by adding SPAN.help_text

Messages

There are 4 styles of messages - Information, Warning, Success and Error. Their structure is as follows:

<p class="message message-warning ">Warning Message</p>
<p class="message message-success ">Success Message</p>
<p class="message message-info message-closable">Informative Message</p>
<p class="message message-error message-closable">Error Message</p>

All notification messages must contain the class .message

To create a closable message, add the class .message-closable

Buttons

<button class="btn-pri"><span>Submit</span></button>

<button class="btn-sec"><span>Submit</span></button>

To create a primary action button, add the class .btn-pri. This button will be the main color of the selected theme.

To create a secondary action button, add the class .btn-sec. This button will be grey.

Layouts

<body class="layout-fixed">	

<body class="layout-fluid">

To change the layout to fluid, add the class .layout-fluid to the body tag. The fixed layout is the default layout, so adding the class .layout-fixed to the body tag is optional.

B) CSS Files and Structure - top

The css file is broken down into groups that all have headings describing which part of the site they style. The outline for the stylesheet is below.

  1. custom.css - main design styles
  2. ie7.css - overrides specific to Internet Explorer 7
  3. ie8.css - overrides specific to Internet Explorer 8

Themes

To enable a specific color style remove the alternate declaration from the LINK tag for that color. Remove any colors you do not wish to use.

  1. red.css - overrides styles in custom.css for red theme
  2. blue.css - overrides styles in custom.css for blue theme
  3. purple.css - overrides styles in custom.css for purple theme
  4. green.css - overrides styles in custom.css for green theme
  5. black.css - overrides styles in custom.css for black theme
  6. orange.css - overrides styles in custom.css for orange theme

C) JavaScript - top

The template uses jQuery for enhancements and effects. You should find these files in the /js/ folder:

  1. jquery.1.4.2.min.js - is the main jQuery file
  2. jquery.flot.min.js - plugin that handles the charting
  3. jquery.cookie.js - plugin that handles cookie functionality
  4. jquery.hoverIntent.min.js - plugin that delays the hover event on elements
  5. jquery.styletoggle.js - handles theme color switching
  6. jquery.lightbox-0.5.min.js - plugin adds image lightbox functionality
  7. jquery.tablesorter.min.js - plugin that add sorting capabilities to data tables
  8. excanvas.js - plugin that adds canvas support to Internet Explorer
  1. azure.js - selects and manages correct theme and layout on page load
  2. azure.chart.js - the code and datapoints used to create the chart and handles the chart redrawing
  3. azure.nav.js - handles the mega dropdown menus
  4. azure.message.js - handles site notifications such as warning messages, success messages, etc.
  5. azure.portlet.js - handles portlet functions
  6. azure.styleswitch.js - handles the enabling/disabling of themes

D) PSD Files - top

In the /psds folder, you can find three .psd files:

  1. header.psd - contains the editable files for the header and navigation elements
  2. portlet-header.psd - containts the editable files for the portlets
  3. arrow-up.psd - defines the up arrows that are used in the portlets
  4. arrow-down.psd - defines the down arrows that are used in the navigation and portlets
